# 基于 PSoC Edge 的分布式多模态巡检集群与边缘 AI 协同优化

## 项目简介
本作品是一个面向工业设备预测性维护的**边缘AI智能巡检终端**。在一块 PSoC Edge E84 开发板上实现了“振动 + 温度 + 声音”三模态实时感知，利用双核异构（M33 + M55）与内置 NPU（Ethos-U55）实现毫瓦级功耗下的毫秒级 AI 推理，并通过“虚实协同”策略，用单块板子模拟多机器人分布式集群的协同确认机制。

## 关键词
- **异构边缘推理**：M33 跑 RT-Thread 管理任务，M55 + NPU 跑 AI 模型，DSP 做 FFT 加速。
- **虚实协同集群**：物理节点与 PC 端 3 个虚拟节点通过串口/MQTT 联动，模拟分布式调度与协同确认。
- **多模态感知融合**：IMU（振动）+ MEMS 麦克风（声音）+ 温度传感器同步采集，融合判决。

## 硬件平台
- **主控**：Infineon PSoC™ Edge E84（双核：M33 + M55，集成 Ethos-U55 NPU）
- **核心传感器**：
  - 振动/姿态：LSM6DS3TR-C（6轴 IMU，I2C 接口）
  - 环境温度：DPS368（I2C 接口）
  - 声音采集：XENSIV™ MEMS 麦克风（PDM 接口）
- **人机交互**：MIPI-LCD 4.3寸屏，LVGL GUI 图形界面
- **通信**：板载 Wi-Fi/BLE（AIROC Wireless）

## 软件架构与关键技术

| 技术点 | 实现方案 |
| :--- | :--- |
| **双核通信** | M33（RT-Thread）与 M55（裸机推理）通过 **rpmsg-lite** 共享内存通信 |
| **NPU 模型部署** | TensorFlow → int8 量化 → **Arm Vela** 编译 → Ethos-U55 推理（< 0.2ms） |
| **DSP 加速** | M55 调用 CMSIS-DSP 库进行加速度频域 FFT 分析 |
| **三级 AI 协同** | NNLite 预检测（µW 级）→ Ethos-U55 深度推理 → 虚拟节点二次协同确认 |
| **低功耗策略** | ULP 模拟监控 + M33 NNLite 唤醒，待机总电流约 450µA（含传感器） |
| **安全启动** | Edge Protect 硬件安全飞地，未签名固件拒绝执行 |

## 核心功能演示
1. **三模态感知**：敲击模拟振动、手指靠近模拟升温、拍手模拟声音，屏幕实时显示波形与数据。
2. **三级 AI 推理闭环**：
   - 一级（M33/NNLite）：微瓦功耗预判异常。
   - 二级（M55/NPU）：毫秒级深度推理，输出置信度（0~100%）。
   - 三级（虚拟节点协同）：低置信度时请求上位机虚拟节点二次确认，融合后决策。
3. **虚实集群联动**：PC 端 Python 脚本模拟 3 台虚拟机器人，接收物理节点的协同请求并回传模拟数据。
4. **低功耗实测**：深度睡眠 ~0.45mA，推理峰值 ~200mA，1000mAh 电池可续航 6 天以上。
5. **安全防护**：演示烧录未签名固件失败，硬件防篡改。

## 编译与烧录注意事项
> ⚠️ **重要**
> - **大 Buffer 落位**：帧缓冲区约 800KB，M55 内部 RAM 仅 256KB，必须通过链接脚本（Linker Script）将大 Buffer 定向到系统 SRAM，**单纯加大堆无效**。
> - **烧录顺序**：必须严格按照 **Secure M33 → NS M33 → M55** 的顺序烧录，顺序错误会导致 M55 核无法启动。
